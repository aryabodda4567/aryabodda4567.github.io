The Analyst Who Stopped Reading Earnings Calls

By The Signal Desk | 6 min read

How one buy-side analyst used a $30 Claude API script to process an entire earnings season and what investment professionals can learn from it.

It Started With a Missed Call

Priya had read fifteen earnings transcripts that day. It was Tuesday, the second week of April, and her coverage list—22 mid-to-large cap industrials and semiconductors—had nearly all reported within a 72-hour window. Her notes were a mess of half-finished sentences. Her coffee was cold. She had a PM meeting in forty minutes.

She almost missed the signal buried in the Caterpillar transcript: a two-sentence aside about dealer inventory drawdowns that contradicted the bullish tone of management's prepared remarks. She caught it at 11:58 PM. Her PM did not get the note until morning.

That night, she started writing a script.

"I'm not trying to replace reading. I'm trying to read better — on the things that actually matter."

The Anatomy of Earnings Overload

The average S&P 500 earnings call transcript runs 6,000-10,000 words. During peak earnings season—roughly three weeks in April and three in July—a portfolio manager or senior analyst with broad coverage can face 30 to 60 transcripts in a single week. At 20 minutes per call (if you're fast), that's up to 20 hours of reading. Add the 10-K, 10-Q, and press release, and the arithmetic gets cruel.

The standard workflow looks like this:

  1. Download transcript (often in PDF from Bloomberg or FactSet)
  2. Ctrl+F for your watchlist terms: guidance, headcount, inventory, free cash flow
  3. Skim Q&A — management's answers to analyst questions often contain more signal than prepared remarks
  4. Summarize findings in notes or a CRM
  5. Repeat 59 times

The inefficiency isn't the reading itself—it's the front-loaded pattern-matching. A skilled analyst knows exactly what she's looking for. The bottleneck is that she has to process thousands of words of boilerplate to get to the 200 words that actually matter.

This is precisely the kind of task large language models are built for.

Building the Pipeline

Priya's solution took about three hours to build and cost less than $30 to run across an entire earnings season. Here's how it works and how you can replicate it.

Step 1 - Ingest the Transcripts

Earnings transcripts are widely available. Bloomberg Terminal, FactSet, and Motley Fool Transcripts are common sources. For this workflow, you can also use the free tier of services like Seeking Alpha or Earnings Whispers. The transcript arrives as plain text or PDF.

Priya's script accepts either format. For PDFs, it uses pdfminer.six to extract raw text. For plain text, it reads directly. The first function is just 12 lines:

import anthropic
import pdfminer.high_level as pdf

def load_transcript(path: str) -> str:
    if path.endswith('.pdf'):
        return pdf.extract_text(path)
    with open(path, 'r', encoding="utf-8") as f:
        return f.read()

# Usage
transcript = load_transcript('NVDA_Q1_2026.pdf')

Step 2 - The Analyst Prompt

This is where the real design work lives. The quality of the output depends entirely on how well you've specified the task. Priya iterated through seven versions of her prompt before settling on one that reliably surfaced actionable signal. The key insight: treat the LLM like a junior analyst who's sharp but has never seen your investment thesis.

ANALYST PROMPTYou are a senior equity research analyst reviewing an earnings call transcript. Analyse the following transcript and return a structured JSON object with these fields: tone_score : integer 0-100 (0-extremely bearish, 50-neutral, 100-extremely bullish) tone_label : one of [Bearish, Guarded, Neutral, Cautiously Optimistic, Positive, Bullish] guidance_delta : brief string describing guidance vs. prior quarter (e.g. '+8% rev guidance raised', 'margin guidance lowered ~150bps') top_themes : list of 3 key themes mentioned (short phrases, not sentences) management_hedge_phrases : list of up to 5 notable hedge/caveat phrases used earnings_surprises : any EPS or revenue beats/misses vs. consensus alert : one of [Upgrade Watch, Long-term Watch, Stable, Watch Margin, Risk Flag] your recommendation for follow-up priority analyst_note : 2-sentence analyst commentary on the single most important non-consensus signal in this transcript Return ONLY valid JSON. No markdown, no preamble.

A few things make this prompt unusually effective for finance workflows:

  • Structured JSON output means the results can be directly piped into a spreadsheet or dashboard—no parsing logic, no cleaning.
  • The tone_score enforces a quantitative discipline. It forces the model to take a position, not hedge.
  • management_hedge_phrases is the sleeper field. It surfaces the specific language executives use to walk back optimism—language that experienced analysts train themselves to hear, but that's easy to skim past.
  • The analyst_note field captures non-consensus signal—and this is where the most interesting outputs tend to surface.

Step 3 - The API Call

The call itself is straightforward. Priya passes the full transcript in the user message (most modern transcripts fit comfortably within Claude's 200K context window):

client = anthropic.Anthropic() # API key from environment

def analyse_transcript(transcript: str) -> dict:
    response = client.messages.create(
        model='claude-opus-20240229',
        max_tokens=1024,
        messages=[
            {
                'role': 'user',
                'content': f'{ANALYST_PROMPT}\n\n---TRANSCRIPT---\n{transcript}'
            }
        ]
    )
    
    import json
    return json.loads(response.content[0].text)

Run this across a batch of transcripts with a simple loop, and you get a structured dataset—one row per company—that can be sorted, filtered, and prioritized in seconds.

"The pipeline doesn't replace the analyst's judgment. It clears the path so the judgment can actually land somewhere."

What It Can't Do

Priya is clear-eyed about the tool's limits—and this is important context for anyone considering something similar.

  • It has no access to consensus estimates unless you pass them in. The guidance_delta field compares current quarter to prior quarter—not to the Street. For that, you'd need to wire in a data API (FactSet, Bloomberg, or similar).
  • Tone scores are not calibrated across companies. An 82 for NVIDIA is not directly comparable to an 82 for a mid-cap utility. Context matters, and the model doesn't have your investment framework unless you give it one.
  • The model can hallucinate specific figures if the transcript is poorly formatted or truncated. Always spot-check numeric claims before acting on them.
  • It is not a compliance-reviewed tool. If your firm has restrictions on sending client data to third-party APIs, that applies here. Use dummy / sanitized transcripts for testing if needed.
